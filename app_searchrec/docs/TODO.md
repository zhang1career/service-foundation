# SearchRec — 产品化与运维待改进点

在保持 **边界清晰的基础抽象与可替换后端** 的前提下，面向真实应用仍需补齐下列能力。与 `docs/searchrec_production_checklist.md` 互补：checklist 偏运维清单，本文偏 **可交付产品与可持续运营**。

## 可观测性与 SLO

- 暴露统一指标：QPS、延迟分位（p50/p95/p99）、错误率、超时率、各适配器（索引/向量/特征）耗时与失败计数。
- 定义并监控 SLO（可用性、查询延迟上限、索引延迟/新鲜度），与告警阈值、值班 runbook 对齐。
- 结构化访问日志与请求 ID，支持按租户/策略/路径抽样与审计。

## 配置与运营面

- 排序权重、策略名、`top_k` 上限、候选倍数等：**版本化配置**与热更新或低停机发布路径；避免仅依赖代码常量。
- **多租户/隔离**：配额、限流、索引与向量集合命名空间，防止单租户拖垮共享后端。
- **降级策略**：在文档中明确「某路召回失败」时的行为（失败返回 vs 部分结果），并与监控联动；与当前 fail-fast 策略取舍一致地产品化说明。

## 检索与相关性（面向业务运营）

- 同义词、停用词、字段级 boost、拼写/查询纠错：在适配器契约或上层暴露 **可控、可审计** 的规则入口（不必内置全部实现，但要有挂载点与版本）。
- 查询分析与慢查询：记录典型查询、空结果率、高延迟查询，便于运营与调参。

## 推荐与效果闭环

- 行为事件（曝光、点击等）的 **摄入规范、幂等与延迟**；与离线训练/评估管道对接的字段约定。
- 离线指标（NDCG、Recall@K 等）与线上 CTR/CVR 的 **定期对照**；实验与基线对比的可重复流程。
- `recommend` 与纯搜索的差异化：长期需 **协同/序列等召回通道** 或外接模型服务时的编排扩展点（保持 API 稳定、内部可插拔）。

## 数据治理与安全

- PII、密钥与特征脱敏；事件与日志保留策略；合规审计字段。
- 索引与向量快照的 **备份、恢复与演练**；配置与密钥轮换流程。

## 发布与实验

- 灰度与回滚：策略/模型/权重变更的 **流量分割**（按用户/租户 cohort）与自动回滚条件。
- A/B 或多臂实验框架与指标看板（可与外部实验平台集成，但需 **稳定分流键与曝光日志** 契约）。

## 文档与开发者体验

- 面向接入方：**SLA 说明、限流与错误码表、各环境依赖（OpenSearch/Milvus 等）的最小权限与容量建议**。
- 故障排查手册：常见错误（适配器不可用、超时、配额）与检查步骤。
